# Spec：新青安 1.0/2.0 資訊、自備款成數選擇、銀行類型比較

- Branch: `feature/xinqingan-2.0-and-bank-types`
- Status: Draft — 待確認後才開始實作
- 對應檔案：`index.html`（單檔 vanilla JS app）

## 背景與決策紀錄

以下三個關鍵設計決策已與使用者確認：

| 決策點 | 選項 | 結果 |
|---|---|---|
| 新青安 2.0 利率精確度 | 單一固定利率 vs 完整 7 年階梯模擬 | **完整 7 年階梯模擬** |
| 民營銀行「兩包利率」定義 | 分段式（前 N 年低利率、之後調高） vs 兩筆完全獨立的貸款額度 | **兩筆完全獨立的貸款額度** |
| 8 大行庫預設清單 | 自訂 vs 標準 8 大公股銀行 | **台灣銀行、土地銀行、合作金庫、第一銀行、華南銀行、彰化銀行、台灣企銀、兆豐銀行** |

## Feature 1：頁首新青安 1.0 / 2.0 資訊卡

### 需求
在 header 下方新增一張資訊卡，說明新青安 1.0（現行）與 2.0（草案，預計 2026/8/1 上路）的差異，供使用者對照。

### 內容（依 2026/7 公開報導整理，草案階段，非最終定案）

**新青安 1.0（現行）**
- 混合式利率：第 1 年 1.745%、第 2 年 1.845%、第 3 年起 1.865%（政府與承貸行庫合計補貼 2 碼，優惠後約落在 1.775% 上下，依銀行牌告微調）
- 貸款上限：1,000 萬元
- 最長年限：40 年
- 寬限期：最長 5 年
- 對象：40 歲以下首購族

**新青安 2.0（草案，預計 2026/8/1 上路）**
- 利率補貼採 7 年階梯式遞減（合計補貼碼數）：
  - 第 1–3 年：內政部 1.5 碼 + 承貸銀行 0.5 碼 = 共 2 碼
  - 第 4 年：內政部 1 碼 + 銀行 0.5 碼 = 共 1.5 碼
  - 第 5 年：內政部 0.5 碼 + 銀行 0.5 碼 = 共 1 碼
  - 第 6–7 年：內政部 0 碼 + 銀行 0.5 碼 = 共 0.5 碼
  - 第 8 年起：補貼歸零，回到基準利率（約 2.275%）
- 貸款上限：1,000 萬元（草案：育有未成年子女家庭可能提高至 1,500 萬）
- 新增排富資格：年收入 200 萬以上不得申貸；房屋總價依縣市高價住宅門檻分級限制；申貸年齡上限 50 歲，且年齡＋貸款年數 ≤ 80
- 寬限期：維持最長 5 年；成數：維持最高 8 成

### UI
- 新增一張 `.card`，標題「🆕 新青安 1.0 vs 2.0」，內容用左右兩欄（`notes-grid` 樣式沿用）並列比較，附上明顯的免責聲明：
  > 以上為 2026/7 媒體報導之草案內容，實際規定以內政部正式公告為準；2.0 相關利率已作為本工具「新青安 2.0 階梯利率」預設值，可於下方自行調整。
- 純資訊展示，不含輸入欄位（實際會影響計算的數字會落在 Feature 4 的「新青安 2.0 階梯利率設定」卡片）。

## Feature 2：自備款改為「成數下拉選單 + 手動輸入」

### 需求
自備款欄位旁新增下拉選單，可選擇「總房價的幾成」自動帶入金額，或選「手動輸入」自由輸入。

### UI / 行為
- `param-group`（自備款）新增 `<select id="downPaymentMode">`，選項：
  - `手動輸入`（預設，維持現況：input 可自由編輯）
  - `一成 (10%)` / `一成五 (15%)` / `二成 (20%)` / `二成五 (25%)` / `三成 (30%)` / `三成五 (35%)` / `四成 (40%)`
- 選擇百分比選項時：
  - `downPayment` input 依 `totalPrice × 百分比` 自動計算並唯讀（灰底、`readonly`），下方 `.unit` 顯示「= XX 萬（總房價 X 成）」
  - 之後若 `totalPrice` 變動，自備款金額連動重算
- 選回「手動輸入」時：input 恢復可編輯，維持選單切換前的最後數值作為起始值。
- 此欄位只影響「現金緩衝」計算，不影響貸款成數/可貸金額（維持現行邏輯：貸款端仍以「貸好貸滿」為假設）。

## Feature 3：銀行類型（8 大行庫 / 一般民營銀行）

### 需求
每個貸款方案卡片新增「銀行類型」選擇，兩種類型呈現不同的欄位與計算邏輯。

### 資料模型變更

```js
// 8 大行庫方案
{
  bankType: 'public8',
  name: '台灣銀行',        // 由預設清單選取，仍可手動修改文字
  rate: 2.3,               // 一般利率（超過新青安額度部分）
  ltv: 80,                 // 最高成數
  years: 30,               // 一般房貸年限
  grace: 3,                // 寬限期（兩筆貸款共用）
  youngLimit: 1000,        // 新青安上限（萬元），可調整（如草案通過育有子女家庭 1500 萬情境）
  yearsY: 40,               // 新青安年限
  // 注意：新青安「利率」不再存於方案內，改讀取全域的 2.0 階梯利率表（Feature 4）
}

// 一般民營銀行方案（兩筆完全獨立貸款）
{
  bankType: 'private',
  name: '陽信銀行',
  grace: 2,                // 寬限期（兩筆貸款共用）
  loanA: { amount: 800, rate: 2.2, years: 30 },  // 額度（萬）/ 利率（%）/ 年限（年）
  loanB: { amount: 300, rate: 2.6, years: 30 },
}
```

### UI
- 方案卡片頂部新增「銀行類型」`<select>`：`8 大行庫` / `一般民營銀行`，切換時重繪該卡片欄位（沿用現有「重繪輸入列會失焦」的教訓 → 只在**切換類型**時重繪整卡，其餘輸入照舊用 `input` 事件局部更新，不重繪 DOM）。
- **8 大行庫**：
  - 「銀行名稱」旁加一個小的預設清單 `<select>`（8 家公股銀行 + 「其他」），選取後帶入 `name` 文字輸入框，文字框仍可自由修改。
  - 欄位：一般房貸利率（%）、最高成數（%）、一般房貸年限（年）、寬限期（年）、新青安上限（萬元）、新青安年限（年）。
  - 新青安「利率」欄位移除（不可編輯），改為顯示唯讀提示：「利率依新青安 2.0 階梯利率表」＋ 一個連結／捲動至 Feature 4 設定卡。
- **一般民營銀行**：
  - 欄位分兩組「貸款 A」「貸款 B」，各自：額度（萬元）、利率（%）、年限（年）。
  - 新增「貸款成數（%）」欄位：預設由 `(loanA.amount + loanB.amount) / totalPrice` 自動帶入，並隨 A/B 額度或總房價變動即時重算；使用者仍可直接手動輸入覆寫（覆寫後不再自動同步，直到再次修改 A 或 B 的額度才會重新自動計算並覆蓋回去 — 與 Feature 2 自備款模式不同，此欄位無「模式切換」，單純「自動預填、可手動覆寫、下次 A/B 變動即重算」）。
  - 共用：寬限期（年）。
  - 不出現新青安相關欄位（不假設民營銀行承作新青安）。
- `addPlan()` 預設新增為 8 大行庫類型（維持現行行為最相近）。

### 計算邏輯變更（`calcRow` / `amortYearly`）
依 `plan.bankType` 分支：

- **`public8`**：
  - `youngLoan = min(price × 80%, plan.youngLimit, bankMax)`（沿用現行公式）
  - `normalLoan = max(bankMax - youngLoan, 0)`
  - 新青安部分改用 Feature 4 的階梯利率表逐年計算（見下方 Feature 4 計算細節），一般部分沿用固定 `plan.rate` 的 PMT。
- **`private`**：
  - `loanA` 與 `loanB` 各自完全獨立：`totalLoan = loanA.amount + loanB.amount`（貸款金額直接輸入，不受成數欄位影響）
  - `ltv` 欄位預設 `= totalLoan / price`，但為可覆寫的獨立欄位（僅供顯示參考／使用者可填銀行實際核貸成數，不回頭影響 `totalLoan`）
  - 寬限期月付 = `loanA.amount × rateA/12 + loanB.amount × rateB/12`
  - 寬限後月付 = `PMT(rateA, (yearsA-grace)×12, loanA.amount) + PMT(rateB, (yearsB-grace)×12, loanB.amount)`

### 表格呈現
主比較表欄位「新青安可貸（萬）」「一般房貸（萬）」改為中性標題「貸款一（萬）」「貸款二（萬）」，並在儲存格內加小標籤：
- `public8` → 貸款一顯示「新青安 2.0」badge，貸款二顯示「一般利率」badge
- `private` → 貸款一顯示「貸款 A」，貸款二顯示「貸款 B」

## Feature 4：新青安月付直接採用 2.0 利率（7 年階梯模擬）

### 需求
新青安月付計算不再用單一可調利率，改成直接套用新青安 2.0 的 7 年階梯式利率（見 Feature 1 表格），且隨貸款年度自動變化。

### 新增全域設定卡片「新青安 2.0 階梯利率設定」
位置：置於 Feature 1 資訊卡與「貸款參數設定」卡片之間。

可編輯的利率表（預設值取自 Feature 1 研究數據，皆可調整）：

| 適用年度 | 預設利率 |
|---|---|
| 第 1–3 年 | 2.025% |
| 第 4 年 | 2.0875% |
| 第 5 年 | 2.15% |
| 第 6–7 年 | 2.2125% |
| 第 8 年起 | 2.275% |

- 這是**全域參數**（`xinqinganV2Schedule`），所有 `public8` 方案共用同一份利率表，不再各自輸入。
- 提供「還原政府草案預設值」按鈕，方便使用者調整後一鍵重設。

### 計算邏輯
- `amortYearly()`：新青安（`public8`）貸款每月依「目前所在年度」查表取得月利率；每當利率換檔（進入新的階梯區間）時，以「當前剩餘本金、剩餘年限」重新計算 PMT（即實務上浮動利率房貸每次調整都會用剩餘本金／期數重新攤提，確保仍在原訂年限內還清）。
- 主比較表兩個月付欄位改為：
  - **首年月付**：寬限期中（或無寬限期則為第 1 個月）以第 1 年利率計算。
  - **長期月付（第 8 年起）**：以新青安補貼完全退場後的基準利率（2.275%，可調）計算，讓使用者了解補貼結束後的月付變化。
  - 原「補助後月付」「負擔比」以**首年**數字為準（貼近使用者近期實際負擔）。
- 年度攤還表（既有功能）新增一欄「新青安當年利率」，讓使用者可以直接在表中看到每年利率變化與對應月付。
- `private` 類型不受影響，維持固定利率 PMT。

### 影響範圍確認
- 主表欄位新增「長期月付（第 8 年起）」，需相應調整 `PLAN_FIELDS`、`renderTable`、`renderBarChart` 圖表資料（長條圖可加第三個資料集，或另開一個「首年 vs 長期」對照圖）。
- `notes` 計算說明區塊需同步更新公式描述。
